# 담당 대리점 지도 (BestRouteFinder)

GA 영업용 담당 대리점(152곳)을 네이버 지도에 표시하고, 목록·검색·필터, 최근접 지하철역, 위촉설계사수별 마커를 제공하는 단일 페이지 웹앱.

## 주요 기능
- 지도보기: 대리점 마커(위촉설계사수별 색·크기), 정보 말풍선, 필터(대리점·본부·지점·위촉인원·호선·역)
- 목록보기: 전체 목록 테이블 + 가까운 역·직선거리
- 목록 업데이트: 엑셀 업로드로 목록 갱신(브라우저에서 파싱)
- 실시간 지오코딩(주소→좌표) + 브라우저 캐시

## 구성 파일
- `index.html` — 배포/호스팅용 실행 파일 (= `agency_map.html` 과 동일 내용)
- `agency_map.html` — 로컬 실행용(동일 앱)
- `_build_template.html` — 데이터 주입 전 템플릿(소스)
- `_build_agencies.json` — 대리점 데이터(소스)
- `_build_stations.json` — 지하철역 좌표(소스, 783개)
- `담당대리점목록.xlsx` — 대리점 원본 데이터
- 문서: `요구사항정의서.md`, `설계안_대리점관리시스템.md`, `설계안2_활동관리시스템_종합설계.md`, `UI목업_활동관리시스템.html`
- `GitHub_업로드_가이드.md` — 배포 방법

## 실행 방법
로컬: 폴더에서 `python -m http.server 5500` 후 브라우저 `http://localhost:5500/`
호스팅: GitHub Pages 등 정적 호스팅에 `index.html` 배포

## 필수 설정 (네이버 지도)
- NCP 콘솔에서 발급한 Maps Client ID 필요(앱 첫 화면에서 입력)
- 접속하는 주소(localhost, github.io 도메인 등)를 NCP **Web 서비스 URL**에 등록해야 지도 인증됨
- 콘솔에서 **Dynamic Map** + **Geocoding** 이용 신청 필요

## 데이터 갱신
- 앱 우측 상단 **"목록 업데이트"** 버튼으로 엑셀 업로드 시 브라우저에 반영·저장(재배포 불필요)
- 소스까지 갱신하려면 `_build_agencies.json`/`담당대리점목록.xlsx` 교체 후 재빌드

## 주의
목록에 연락처·주소(개인/업무 정보)가 포함됨. 공개 저장소·공개 호스팅 시 노출에 유의.
