## 1. 비전 & 목표

지도/목록 조회 앱을 넘어, **"활동 계획 → 현장 활동 → 이력 축적 → 실적 비교 → 다음 계획 개선"** 의 선순환 루프를 도는 영업 활동 관리 시스템으로 발전시킨다.

핵심 가치 3가지:
1. **기록의 자산화** — 방문·교육·소식지·판촉물·식사/간식 등 모든 활동을 구조화해 이력으로 축적
2. **활동–실적 인과 가시화** — 어떤 활동이 신계약 실적으로 이어지는지 비교·분석
3. **스마트 계획** — 동선·대리점 인원·과거 활동 이력·실적을 함께 고려해 최적의 활동 계획 자동 추천

## 2. 핵심 개념 모델 (선순환 루프)

```
[계획 수립] → [현장 활동/방문] → [활동 기록] → [이력 축적]
     ↑                                              │
     └──────── [활동–실적 분석/인사이트] ←──── [신계약 실적 연동]
```

각 단계가 데이터를 남기고, 분석 결과가 다음 계획의 입력이 된다(피드백 루프).

## 3. 모듈 구성

| # | 모듈 | 역할 |
|---|---|---|
| M1 | 대리점 마스터 | 대리점/지점 정보, 위촉설계사수, 위치/권역, 등급 (현행 확장) |
| M2 | 활동 계획 | 기간별 방문·활동 계획 수립, **최적 계획 추천**, 동선 시각화 |
| M3 | 활동 기록 | 방문 결과·활동내역 입력(모바일 우선), 활동유형·비용·메모·사진 |
| M4 | 실적 연동 | 신계약 실적(건수/보험료/수수료) 대리점·기간별 적재 |
| M5 | 분석/대시보드 | 활동–실적 비교, 활동유형별 기여/ROI, 알림·인사이트 |
| M6 | 최적계획 엔진 | 동선+인원+이력+실적 기반 우선순위·동선·추천활동 산출 |

## 4. 데이터 모델 확장 (초안)

- **agencies** (M1): id, 지점코드(고유), 대리점명, 본부명, 지점명, 위촉설계사수, 등급, 주소, 위도/경도, 권역(구), 개시일
- **activity_types** (M3): 코드/명 — 교육, 소식지, 판촉물, 식사, 간식, 기타 (확장 가능)
- **activities** (M3): id, 대리점_id, 담당자_id, 일시, 유형(다중), 비용, 면담자, 메모, 사진, 결과/후속, 계획_item_id(있으면)
- **plans** (M2): id, 담당자_id, 기간, 상태(초안/확정/완료)
- **plan_items** (M2): id, plan_id, 대리점_id, 예정일, 추천활동유형, 예상 이동시간, 순서
- **performance** (M4): id, 대리점_id, 기간(월/주), 신계약 건수, 보험료, 수수료, 채널
- **users** (담당자): id, 이름, 역할(관리자/편집/조회)
- **(파생)** 대리점별 활동요약, 최근방문경과일, 활동–실적 지표

## 5. 최적 활동계획 엔진 (M6) — 핵심 차별화

**입력**
- 동선: 대리점 위/경도 + 이동시간(Naver Directions)
- 규모: 위촉설계사수(클수록 우선순위 가중)
- 과거 활동 이력: 최근 방문 경과일, 활동 빈도/유형, 마지막 교육/소식지 시점
- 실적: 최근 신계약 추세, 활동 대비 실적 반응도, 미개척/하락 신호

**우선순위 점수 (대리점별)**
```
P = w1·규모 + w2·방문경과일 + w3·실적잠재력 + w4·활동반응도 − w5·과방문패널티
```
(가중치 w는 운영하며 조정. 초기엔 규칙 기반, 데이터 쌓이면 학습 기반으로 고도화)

**제약**
- 하루 가용시간(예: 6시간), 방문당 소요(이동+면담), 주당 목표 방문 수, 권역 묶음

**알고리즘 흐름**
1. 점수로 후보 대리점 선별 → 2. 권역(구) 기준 클러스터링 → 3. 하루 단위로 시간예산 내 배정 → 4. Naver Directions로 이동시간 최소 순서(근사 TSP) → 5. 각 방문에 **추천 활동유형** 부여

**추천 활동유형 로직(예)**
- 교육 미시행 90일↑ → "교육" 제안 / 소식지 정기주기 도래 → "소식지" / 실적 하락 → 집중 케어(식사·상담) / 신규/저활동 → 관계형성(간식·인사)

**출력**: 주간 계획(요일별 대상·순서·예상 이동시간·추천활동) + 지도 동선. 사용자가 드래그로 조정 가능.

## 6. 활동–실적 분석 (M5)

**지표**: 활동강도(방문수·교육횟수·판촉비) vs 신계약(건수·보험료) — 대리점·기간 단위

**분석 방법**
- 상관/회귀: 활동량 ↔ 신계약 관계
- 활동유형별 기여: 어떤 활동이 실적과 더 연결되는가
- 시차 분석: 활동 후 N주 실적 변화(효과 발현 시점)
- ROI: 활동비용 대비 신계약 수수료
- 코호트 비교: 교육 시행 vs 미시행 대리점 실적 격차

**인사이트 카드(예)**
- "최근 4주 미방문 대리점 23곳 — 평균 신계약 12%↓"
- "교육 시행 대리점, 시행 후 6주 신계약 +18%"
- "판촉물 ROI 상위 권역: 강남·서초"

## 7. UI / 화면 구성 (목업 `UI목업_활동관리시스템.html` 참조)

좌측 글로벌 내비 + 상단 컨텍스트 바 기반. 핵심 6화면:

1. **대시보드(홈)** — 이번달 KPI(방문·교육·신계약·활동대비실적지수), 방문 필요 알림, 이번주 추천 동선, 활동 vs 실적 미니 산점도
2. **지도(현행 확장)** — 마커 색=실적/활동상태, 클릭 시 활동이력 요약 + "방문 기록" 바로가기
3. **활동 계획 수립** — 좌: 추천 대상(점수·사유 태그) / 중앙: 지도 동선 / 우: 요일별 일정(드래그). "자동 계획 생성" 버튼, 예상 이동시간·방문수
4. **활동 기록(모바일 우선)** — 대리점·날짜·활동유형 멀티선택(교육/소식지/판촉물/식사/간식/기타)·비용·면담자·메모·사진·결과
5. **대리점 상세** — 프로필(인원/등급/위치/가까운역) + 활동 타임라인 + 신계약 추이 + 다음 추천 액션
6. **분석 대시보드** — 활동 vs 신계약 산점도, 활동유형별 ROI, 대리점 리더보드, 인사이트 카드

디자인 톤: 현행 네이버 그린 계열 유지, 모바일 반응형(현장 입력 고려), 큰 터치 타깃·빠른 입력 폼.

## 8. 단계적 로드맵

- **P0 데이터 분리 + DB 이관**(인프라 설계안 참조) — 기반 마련
- **P1 활동 기록(M3) + 대리점 상세(M1)** — 이력 축적 시작(가장 먼저 가치)
- **P2 실적 연동(M4) + 분석 대시보드(M5)** — 활동–실적 비교
- **P3 활동 계획(M2)** — 수동 계획 + 동선 시각화
- **P4 최적계획 엔진(M6)** — 점수·동선 자동 추천(규칙 기반 → 학습 기반)
- **P5 모바일 PWA·고도화** — 현장 사용성, 알림, 리포트 자동화

원칙: 이력 데이터가 먼저 쌓여야 분석·추천이 의미 있으므로 **기록(P1)부터** 시작.

## 9. 가정 & 다음 결정사항

**가정**: 신계약 실적 데이터는 별도 시스템/엑셀에서 대리점·기간 단위로 확보 가능. 담당자는 영업 트레이너(소수). 모바일은 반응형 웹(PWA)으로 우선 대응.

**결정 필요**: ① 실적 데이터 소스/주기 ② 활동유형·비용 항목 확정 ③ 가중치 초기값(규모·경과일·실적) ④ 권한 정책 ⑤ 우선 구현 모듈(P1 기록 권장)
