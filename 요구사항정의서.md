## 1. 개요 / 목적

담당 대리점 155곳의 주소를 네이버 지도 위에 표시하고, 영업 방문 시 최적 이동 경로를 계획할 수 있는 웹 서비스를 만든다. GA 영업 동선 관리·방문 효율화가 목적이다.

## 2. 범위

- 대상 데이터: `담당대리점목록.xlsx` (서울 11개 구, 155개 지점)
- 플랫폼: 네이버 지도 API v3 (Web Dynamic Map + Geocoding 서브모듈)
- 산출물 형태: 단일 HTML 파일 (서버 불필요, 브라우저에서 실행)

## 3. 기능 요구사항 (구현 완료)

- **지도 표시**: 서울 중심 지도, 155개 대리점 위치 마커 표시
- **실시간 지오코딩**: 주소 → 좌표 변환을 브라우저에서 직접 수행 (geocoder 서브모듈)
- **마커 정보창**: 지점명, 대리점(본부), 주소, 연락처(클릭 전화 연결), 위촉설계사 수, 개시일자
- **필터/검색**: 대리점(본부)별, 지역(구)별 필터 + 지점명·주소 텍스트 검색
- **사이드바 리스트**: 조건에 맞는 지점 목록, 클릭 시 해당 마커로 이동 + 정보창 표시
- **좌표 캐시**: 변환 결과를 브라우저(localStorage)에 저장 → 재방문 시 즉시 표시, API 호출 최소화. "캐시 초기화" 버튼 제공
- **API 키 관리**: 화면에서 키 입력·변경·삭제 (localStorage 저장)
- **변환 실패 표시**: 좌표 변환 실패한 주소는 목록에 별도 표시

## 4. 비기능 요구사항 (구현 완료)

- **인증 호환성**: 구 콘솔 키(`ncpClientId`)와 신규 Maps 콘솔 키(`ncpKeyId`)를 자동으로 모두 시도
- **안정성**: 지오코딩 1건당 10초 타임아웃 → 무응답이어도 멈추지 않고 진행
- **성능**: 동시 5건 병렬 지오코딩 (155개 약 10~20초)
- **로딩 순서 대응**: geocoder 서브모듈이 지도 본체보다 늦게 로드되는 경우 준비될 때까지 대기
- **단일 파일**: 외부 의존 없이 HTML 한 개로 동작

## 5. 기술 스택 / 아키텍처

- 네이버 지도 API v3 (`https://oapi.map.naver.com/openapi/v3/maps.js`)
- 데이터는 빌드 시 HTML 안에 JSON으로 주입 (`_build_template.html` + `_build_agencies.json` → `agency_map.html`)
- 상태/캐시: 브라우저 localStorage

## 6. 데이터 명세

원본 `담당대리점목록.xlsx`에서 다음 필드 사용: 대리점명, 본부명, 지점명, 개시일자, 위촉설계사 수, 연락처, 주소. 주소 앞 우편 구분 코드(예: `(031-16)`)는 지오코딩 정확도를 위해 제거 후 사용.

## 7. 향후 요구사항 (백로그)

- **최적 이동 경로(핵심 미구현)**: 네이버 Directions API는 한 번에 경유지 5~15개 제한 → "지역/본부 필터 → 그날 방문할 5~15곳 선택 → 최적 순서·경로 계산" 방식으로 구현 예정
- **API 키 통일**: Directions가 포함된 신규 키(SalesRouteFinder)로 통일 권장 (해당 앱에 Dynamic Map API + Web 서비스 URL 추가 필요)
- **지하철 출구 데이터 번들**: 공공데이터 "도시철도 역사 출입구" 좌표 파일을 받아 `SUBWAY_EXITS`에 주입 → 목록보기의 역/출구/도보거리 자동 표기 (도보거리는 직선거리 기반 추정)
- (검토) 방문 일정 저장, 경로 인쇄/내보내기, 모바일 화면 대응

## 9. 운영 메모

- **로컬 실행**: BestRouteFinder 폴더에서 `python -m http.server 5500` 실행 후 브라우저에서 `http://localhost:5500/agency_map.html` 접속 (file:// 직접 열기는 인증 실패)
- **콘솔 설정**: 사용하는 앱에 Web Dynamic Map + Geocoding 활성화, Web 서비스 URL에 접속 주소(예: `http://localhost:5500`) 등록
- **보유 키**: airealtor(구 콘솔, 현재 동작), SalesRouteFinder(신규 콘솔, Directions 포함 — 경로 기능용 권장, Dynamic Map 추가 필요)
